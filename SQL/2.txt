2) where 절
: 조건절. 원하는 데이터를 가져올 때 사용 

3) order by 절
: 데이터를 순서대로 정렬할 때 사용 
 기본적으로 오름차순 정렬. 내림차순 정렬시 desc를 사용 descending

4) limit
: 조회하는 데이터의 갯수를 지정할 때 사용 
 order by와 연결하여 상위 몇개 또는 하위 몇개 조회할 때 주로 사용

5) distinct
: 데이터를 조회할 때 중복 데이터를 배고 한 번만 보이고자 사용 
 주의, 단순히 데이터의 중복을 빼는 역할이며 group 그룹과는 다른 개념!! 

<연산자>
1) 산술연산자
 + - * / %

2) 비교연산자
 >=, <=, >, <, =, !=, <>
 같지 않다는 의미의 <> 기억 필요 !=와 같음 

3) 논리연산자
 AND, OR, NOT

4) 집합연산자
 UNION : 여러 테이블을 하나로 합칠 때 사용하나, 별로 사용하지 않음 

5) IS NULL, IS NOT NULL
 알 수 없는 값(Unknown Value)의 의미. " 빈 문자열과는 다른 개념 "

6) IN 
 OR 연산자를 수행해야하는 경우 좀 더 간단하게 사용할 수 있음

7) BETWEEN AND
 ~이상, ~이하라는 조건을 수행할 수 있음 

8) LIKE
 문자열의 패턴을 지정할 수 있음
 % 0개 이상의 문자를 가진 문자열을 포함한다는 의미 ( 0개일 수도,여러개일 수도 있음)
 _ 정확히 1개의 문자를 포함한다는 의미

 SQL의 특징
- 쉽고 직관적임
= 거의 모든 관계형 데이터베이스이 표준 언어
- 데이터를 다루는 다양한 가능 제공

SQL문의 종류
1)DDL(Data Definition Language) 데이터 정의어
: 데이터베이스의 구조를 정의하거나 변경하는데 사용 
-create: 테이블,인덱스,뷰를 생성함
-alter: 테이블 구조를 변경함
-drop: 테이블, 인덱스, 뷰등을 삭제함
-truncate: 테이블의 모든 데이터를 삭제하지만 구조는 유지 

2)DML(Data Manipulation Language) 데이터 조작어
: 데이터 행(row)를 조작,삽입,엡데이트,삭제하는데 사용됨
- selec: 데이터를 조회
-insert: 데이터를 삽입
-update: 데이터를 수정
-delete: 데이터를 삭제
3)DCL(Data Control Language) 데이터 제어어
: 데이터에 대한 액세스를 제어하고 권한을 부여 또는 취소하는데 사용됨
- grant: 사용자의 권한 부여
-revoke: 사용자의 권한 취소
4)TCL(Transaction Control Language) 트랜잭션 제어어
: 트랜잭션의 수행 결과를 저장하거나 또는 취소하는데 사용됨
- commit: 트랜잭션 작업 내용을 데이터베이스에 영구 저장
- rollback: 트랜잭션 작업 내용을 취소
