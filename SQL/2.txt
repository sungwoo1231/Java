SELECT 문
테이블내에 저장되어 있는 데이터를 조회하기 위해 사용하는 명령어 

1)기본형식
컬럼: ex)고객회사명,담당자명,직위 등등 가로줄
데이터:세로줄 갯수
as: 이름 바꾸기 ex)마일리지 as 포인트 -> 마일리지 -> 포인트로 
and:
order:정렬순서
limit: 갯수제한

select 컬럼명1, 컬럼명2 as 별명 from 테이블명 
where 조건 
order by 컬럼명
limit 숫자 

2) where 절
: 조건절. 원하는 데이터를 가져올 때 사용 

3) order by 절
: 데이터를 순서대로 정렬할 때 사용 
 기본적으로 오름차순 정렬. 내림차순 정렬시 desc를 사용 descending

4) limit
: 조회하는 데이터의 갯수를 지정할 때 사용 
 order by와 연결하여 상위 몇개 또는 하위 몇개 조회할 때 주로 사용

5) distinct
: 데이터를 조회할 때 중복 데이터를 배고 한 번만 보이고자 사용 
 주의, 단순히 데이터의 중복을 빼는 역할이며 group 그룹과는 다른 개념!! 

<연산자>
1) 산술연산자
 + - * / %

2) 비교연산자
 >=, <=, >, <, =, !=, <>
 같지 않다는 의미의 <> 기억 필요 !=와 같음 

3) 논리연산자
 AND, OR, NOT

4) 집합연산자
 UNION : 여러 테이블을 하나로 합칠 때 사용하나, 별로 사용하지 않음 

5) IS NULL, IS NOT NULL
 알 수 없는 값(Unknown Value)의 의미. " 빈 문자열과는 다른 개념 "

6) IN 
 OR 연산자를 수행해야하는 경우 좀 더 간단하게 사용할 수 있음

7) BETWEEN AND
 ~이상, ~이하라는 조건을 수행할 수 있음 

8) LIKE
 문자열의 패턴을 지정할 수 있음
 % 0개 이상의 문자를 가진 문자열을 포함한다는 의미 ( 0개일 수도,여러개일 수도 있음)
 _ 정확히 1개의 문자를 포함한다는 의미

<쿼리실행(완료)순서>
1. from 어느 테이블에서 정보를 가져올지 결정
2. where 어떤 정보를 가져올지 결정(조건)
3. group by 그룹화
4. selcet 원하는 컬럼(열)을 추출,생성 - 별칭 생성
5. having 그룹에 조건을 적용
6. order by 정렬
7. limt 개수 정렬 

<서브쿼리의 경우에는 where절에서도 별칭 사용가능>
selcet *
from (
       select datediff(발송일,요청일) as 지연일수
from 주문)
where 지연일수 >= 7;
